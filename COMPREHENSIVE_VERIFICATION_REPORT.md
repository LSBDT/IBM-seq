### 元のバージョン (ORIGINAL)
- **ノード数**: 27（同じ数だが、**異なるノード集合**）
- **エッジ数**: **0**（間違ったノード集合のため）

### なぜこうなるのか

**元のバージョンの処理**:
```
1. membershipからcommunity_id=1669のノード名を取得
   → しかし、これらのノード名には間違ったcommunity_idが付与されている
   → 実際にはcommunity_id=1669に属していない27個のノード名

2. induced_subgraph(graph, これらの27個のノード)
   → graphから27個のノードを抽出
   → これらのノードは実際には異なるコミュニティに属している
   → 互いにエッジがない

3. 結果:
   - エッジ数 = 0
   - 密度 = 0 / max_edges = 0
   - 連結性 = FALSE
   - Diameter = NA
   - 次数 = 全て0
```

## 7. 修正の詳細

### 7.1 変更内容

**ファイル**: `pipeline/r_script/02_clustering.R`
**行番号**: 59-60行目

**修正前**:
```r
df$vertices <- merge(df$vertices, as.data.frame(test_wide),
                     by.x = "name", by.y = "to", all.x = TRUE)
```

**修正後**:
```r
df$vertices <- merge(df$vertices, as.data.frame(test_wide),
                     by.x = "name", by.y = "to", all.x = TRUE, sort = FALSE)
```

### 7.2 修正の効果

- ✅ ノード順番が保持される（3531/3531一致）
