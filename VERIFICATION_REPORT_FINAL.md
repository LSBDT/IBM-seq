## 4. 修正の詳細

### 4.1 変更箇所

**ファイル**: `pipeline/r_script/02_clustering.R`
**行番号**: 59-60行目

**修正前**:
```r
df$vertices <- merge(df$vertices, as.data.frame(test_wide),
                     by.x = "name", by.y = "to", all.x = TRUE)
```

**修正後**:
```r
# CRITICAL: sort = FALSE を指定してノード順番を保持
#   merge() はデフォルトでソートするため、graph の頂点順番と不一致になる
#   → 後続の cbind(df$vertices, clusters, community_id) でミスマッチが発生
df$vertices <- merge(df$vertices, as.data.frame(test_wide),
                     by.x = "name", by.y = "to", all.x = TRUE, sort = FALSE)
```

**変更内容**: `sort = FALSE`パラメータを追加

### 4.2 なぜこの修正が必要だったのか

**問題の連鎖**:

1. **48行目**: `df <- igraph::as_data_frame(graph, "both")`
   - `df$vertices`はgraphの頂点順番と一致

2. **59-60行目**: `merge()`で結合（デフォルト: `sort = TRUE`）
   - `df$vertices`がアルファベット順にソートされる
   - graphの頂点順番とは無関係に

3. **71行目**: `clusters <- components(graph)$membership`
   - `clusters`はgraphの頂点順番に対応

4. **135-141行目**: `cbind(df$vertices, clusters, community_id)`
   - ソート済みの`df$vertices`（アルファベット順）
   - graphの順番の`clusters`と`community_id`
   - **異なる順番のデータを横に並べてしまう**

**結果**: position iのノード名とcommunity_idが完全にずれる

### 4.3 修正の効果

`sort = FALSE`を追加することで:
- `df$vertices`の行順番が保持される
- graphの頂点順番と一致したまま
- `cbind()`で正しく結合される
- community_idとノード名が正しく対応する

## 7. SMモードへの影響

**結論**: SMモード自体は問題の原因ではありません。

`merge()`のソート問題は、**SMモードの有無に関わらず発生**しています：

- `tmp/sm/` (SMモード): order_identical = FALSE
- `tmp/` (通常モード): order_identical = FALSE
- `tmp/test_no_rds/`: order_identical = FALSE

→ 全てのバージョンで同じ問題が発生していました。
